# 캠핑 trip 분석 프롬프트

아래 입력 데이터와 문서 규칙을 바탕으로 특정 trip에 대한 캠핑 분석 결과를 Markdown으로 작성한다.

출력 규칙:

- Markdown 본문만 출력한다.
- 사용자가 다른 언어를 명시적으로 요구하지 않는 한 한국어 Markdown으로 출력한다.
- 최상단 제목은 `# <trip 제목> 캠핑 분석 결과` 형식을 사용한다.
- 아래 섹션을 순서대로 포함한다.

```md
## 1. 요약
## 2. 추천 장비
## 3. 연령대별 개인 준비물
## 4. 부족한 장비/소모품
## 5. 출발 전 체크리스트
## 6. 추천 식단/요리
## 7. 이동 중 방문 후보 지역과 장소
## 8. 캠핑장 주변 방문 후보 지역과 장소
## 9. 캠핑장 tip
## 10. 주의사항 및 한계
## 11. 다음 캠핑 추천
```

분석 규칙:

- 날짜와 장소가 모두 있으면 가장 구체적으로 추천한다.
- 날짜만 있으면 계절성과 기온 중심의 범용 추천을 제공하고 장소 한계를 밝힌다.
- 장소만 있으면 장소 특성 중심 추천을 제공하고 날짜 한계를 밝힌다.
- 어린 동행자가 있으면 보온, 여벌 옷, 휴식, 비상약을 더 보수적으로 본다.
- 밤 기온이 낮으면 보온 장비와 따뜻한 음식 비중을 높인다.
- 우천 가능성이 있으면 우비, 방수 수납, 여벌 옷, 실내 대체 코스를 우선 제안한다.
- 소모품이 부족하거나 사전 점검 상태가 불량이면 경고를 포함한다.
- 차량 적재량 정보가 부족하면 적재 관련 판단 한계를 명시한다.
- 불필요한 과잉 장비는 제외 후보로 짧게 언급한다.
- 7, 8번 섹션은 장소 유형만 적지 말고 `지역명 + 구체 장소명` 형식으로 작성한다.
- 7, 8번 각 추천 항목에는 방문 타이밍, 동선 이유, 아이 동반 적합성 같은 실용 이유를 짧게 포함한다.
- `cache/campsite-tips/*.json` 에 `found` 상태의 후기 조사 결과가 있으면 9번 섹션에 `실제 이용 팁 + 왜 도움이 되는지` 형식으로 2~4개 정리한다.
- 9번 섹션 팁은 후기 조사 JSON의 `tip_items` 와 `summary` 를 바탕으로만 작성하고, 없는 내용을 추측으로 추가하지 않는다.
- 후기 조사 JSON에 `best_site_items` 가 있으면 9번 섹션 안에 `명당 후보` 항목을 따로 두고, `사이트 번호/구역 + 이유 + 있으면 주의점` 형식으로 정리한다.
- `best_site_items` 는 후기에서 실제로 언급된 사이트 번호나 구역만 사용하고, 번호를 추정해서 만들지 않는다.
- 9번 섹션 마지막에는 후기 조사 JSON의 `sources` 가 있으면 `참고 후기` 형식으로 1~3개까지 짧게 덧붙일 수 있다.
- 후기 조사 결과가 없거나 `not_found`/`failed` 상태면 9번 섹션에 `같은 캠핑장 후기 기반 tip을 충분히 확인하지 못했다` 는 한계를 짧게 적는다.
- 11번 섹션은 현재 trip 이후 1개월 이내 주말 후보를 기준으로 추천한다. 현재 trip 날짜가 있으면 종료일 다음 날부터 계산하고, 날짜가 없으면 제공된 `next-camping-recommendation-context` 기준일을 사용한다.
- 11번 섹션에는 추천 시기 1~2개와 계절에 잘 맞는 가족 단위 장소 후보 1~2개를 함께 적는다.
- 11번 섹션 각 추천에는 계절 적합성, 가족 적합성, 다시 확인할 정보(날씨, 예약, 아이 편의시설 등)를 짧게 포함한다.
- `links.yaml` 또는 `next-camping-recommendation-context.recommended_links` 가 있으면 확인용 링크를 1~3개까지 짧게 덧붙일 수 있다.
- 장소 캐시나 `requested_stops` 가 없어도 지역명, 캠핑장명, 출발지, 동행 조건을 바탕으로 후보 장소를 추론할 수 있다.
- 실제 장소명을 자신 있게 특정하기 어렵거나 외부 근거가 약하면 `후보` 또는 `추정` 이라고 밝히고 근거 부족을 함께 적는다.
- 7, 8, 11번 섹션 모두 실제 장소명 특정이 어렵다면 `후보` 또는 `추정` 성격을 드러낸다.

문체 규칙:

- 실용적이고 간결하게 작성한다.
- 섹션마다 너무 긴 서술 대신 준비 항목과 이유를 짝으로 제시한다.
- 추정은 추정이라고 밝힌다.
- 사용자가 다른 언어를 명시적으로 요구하지 않는 한 제목, 본문, 불릿을 모두 한글로 작성한다.

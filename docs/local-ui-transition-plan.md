# 로컬 UI 전환 계획

## 1. 문서 목적

이 문서는 기존 `trip 분석 중심 UI` 를 `운영형 로컬 캠핑 관리자 UI` 로 확장한 결과와 현재 기준을 정리한다.

## 2. 전환 방향

기존 구조:

- trip 목록
- trip 상세
- 분석 결과

현재 구조:

- 대시보드
- 사람 관리
- 차량 관리
- 장비 관리
- 캠핑 계획
- 캠핑 히스토리
- 외부 링크
- 카테고리 설정
- 보조 설명

## 3. 유지한 원칙

- 로컬 우선 구조
- `.camping-data/` 운영 데이터 저장
- 사람이 관리하는 YAML 입력
- Markdown 결과 저장
- 브라우저는 로컬 API만 호출
- 개인 준비물은 AI가 파생하는 결과라는 도메인 규칙 유지

## 4. 추가된 UI 원칙

- 첫 화면은 `대시보드`
- 메뉴는 항상 드러나야 한다
- 사람과 차량은 계획 화면이 아니라 준비 데이터 화면에서 관리한다
- 계획 페이지는 `폼 + AI 보조 + 결과` 구조를 유지하되 선택형 입력으로 단순화한다
- 분석 실행은 동기 대기가 아니라 백그라운드 상태 표시와 결과 자동 갱신 흐름으로 다룬다
- 히스토리는 계획 종료 후 아카이브로 생성하고 당시 사람/차량 정보를 스냅샷으로 함께 저장한다
- 경로와 생성 규칙은 `보조 설명` 화면으로 분리한다

## 5. 현재 완료 범위

- 메뉴형 레이아웃 적용
- 사람 관리 화면 추가
- 차량 관리 화면 추가
- 장비 CRUD 화면 추가
- 장비 카테고리 셀렉트와 카테고리 설정 화면 추가
- 계획 CRUD 및 분석 화면 확장
- 계획 화면의 백그라운드 분석 상태 조회와 자동 폴링 추가
- 같은 계획 중복 분석 방지와 `분석 중...` 버튼 상태 유지 추가
- 분석 중 계획 삭제/히스토리 이동 비활성화 추가
- 계획 화면의 동행자/차량 선택형 입력 적용
- 히스토리 목록/수정/삭제와 저장된 결과 열기 화면 추가
- 히스토리에 동행자/차량 스냅샷 표시 추가
- 외부 링크 CRUD 및 카테고리 그룹 화면 추가
- 보조 설명 화면 추가

## 6. 남은 고도화 후보

- 사람/차량 검색과 정렬 강화
- 장비 카테고리 필터와 검색 강화
- 히스토리 비교 보기
- 외부 링크 정렬/즐겨찾기
- AI 보조 제안의 정밀도 향상

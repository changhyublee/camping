# 로컬 UI 전환 계획

## 1. 문서 목적

이 문서는 기존 `trip 분석 중심 UI` 를 `운영형 로컬 캠핑 관리자 UI` 로 확장한 결과와 남은 기준을 정리한다.

## 2. 전환 방향

기존 구조:

- trip 목록
- trip 상세
- 분석 결과

현재 구조:

- 대시보드
- 장비 관리
- 캠핑 계획
- 캠핑 히스토리
- 외부 링크

## 3. 유지한 원칙

- 로컬 우선 구조
- `.camping-data/` 운영 데이터 저장
- 사람이 관리하는 YAML 입력
- Markdown 결과 저장
- 브라우저는 로컬 API만 호출
- 개인 준비물은 AI가 파생하는 결과라는 도메인 규칙 유지

## 4. 추가된 UI 원칙

- 첫 화면은 `대시보드`
- 메뉴는 항상 드러나야 한다
- 계획 페이지는 `폼 + AI 보조` 구조를 사용한다
- AI 제안은 자동 저장하지 않는다
- 히스토리는 계획 종료 후 아카이브로 생성한다

## 5. 현재 완료 범위

- 메뉴형 레이아웃 적용
- 장비 CRUD 화면 추가
- 계획 CRUD 및 분석 화면 확장
- 히스토리 목록/수정/삭제와 저장된 결과 열기 화면 추가
- 외부 링크 CRUD 및 카테고리 그룹 화면 추가

## 6. 남은 고도화 후보

- 동행자 선택 UI 개선
- 장비 카테고리 필터와 검색 강화
- 히스토리 비교 보기
- 외부 링크 정렬/즐겨찾기
- AI 보조 제안의 정밀도 향상

# MVP 범위

## 1. 문서 목적

이 문서는 현재 구현 기준의 로컬 운영형 캠핑 관리자 범위를 정의한다.

## 2. 현재 MVP 정의

현재 MVP는 아래를 만족하는 로컬 도구다.

- 장비를 CRUD 로 관리할 수 있다
- 캠핑 계획을 작성/수정/삭제할 수 있다
- AI 보조와 분석 결과를 받을 수 있다
- 완료된 계획을 히스토리로 아카이브할 수 있다
- 외부 링크를 북마크 형태로 관리할 수 있다

## 3. MVP 필수 기능

- 대시보드
- 장비 관리 CRUD
- 캠핑 계획 CRUD
- 계획 검증과 분석 실행
- 결과 저장
- 히스토리 조회/수정/삭제
- 외부 링크 CRUD

## 4. 중요한 도메인 원칙

- 개인 준비물은 직접 관리하는 목록이 아니라 AI 추천 결과다
- 브라우저에서 OpenAI API를 직접 호출하지 않는다
- 운영 데이터는 계속 `.camping-data/` 에 저장한다
- 히스토리는 계획과 분리된 파일로 관리한다

## 5. 현재 제외 항목

- 원격 API 서버
- 데이터베이스 서버
- 로그인/인증
- 다중 사용자 협업
- 모바일 앱
- 외부 API 자동 수집의 필수화
- 개인 준비물 수동 인벤토리 관리자

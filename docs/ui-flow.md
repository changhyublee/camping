# UI 흐름

## 1. 문서 목적

이 문서는 현재 로컬 웹 UI의 주요 화면과 사용자 흐름을 정의한다.

## 2. 기본 원칙

- 첫 화면은 `대시보드`
- 메뉴는 `운영 허브 / 준비 데이터 / 관리 설정` 세 그룹으로 나눈다
- 계획 페이지는 `계획 목록 + 원본 입력 + AI 보조/결과` 를 한 화면에서 다룬다
- 사람과 차량은 계획에서 직접 입력하지 않고 준비 데이터 화면에서 관리한 뒤 선택한다
- AI 제안은 자동 반영하지 않는다

## 3. 주요 화면

### 화면 1. 대시보드

- 운영 현황 카드
- 예정 캠핑 목록
- 최근 히스토리 목록
- 빠른 이동 버튼

### 화면 2. 사람 관리

- 사람 목록
- 이름, 연령대, 출생연도 입력
- 건강 특이사항, 필수 복용약, 민감도 수정

### 화면 3. 차량 관리

- 차량 목록
- 표시 이름과 설명
- 탑승 인원, 적재량, 차량 메모 수정

### 화면 4. 장비 관리

- 반복 장비 탭
- 소모품 탭
- 출발 전 점검 탭
- 카테고리 그룹 기본 접힘
- 아이템 상세 기본 접힘
- 목록 편집
- 항목 추가
- 카테고리 셀렉트 선택
- 반복 장비 상세 패널에서 모델명, 구매 링크, AI 메타데이터, 재수집 버튼 표시

### 화면 5. 캠핑 계획

- 계획 목록
- 계획 폼 편집
- 등록된 동행자 다중 선택
- 등록된 차량 단일 선택
- 선택된 사람/차량 요약
- AI 보조 대화
- 분석 실행
- 결과 Markdown
- 히스토리 아카이브

### 화면 6. 캠핑 히스토리

- 히스토리 목록
- 저장된 결과 Markdown 열기
- 당시 동행자/차량 스냅샷 확인
- 메모 수정
- 삭제

### 화면 7. 외부 링크

- 카테고리별 링크 목록
- 카테고리 그룹 안에서 바로 수정
- 링크 추가/수정/삭제

### 화면 8. 카테고리 설정

- 장비 섹션별 카테고리 목록
- 카테고리 편집 카드 기본 접힘
- 카테고리 표시 이름 수정
- 새 카테고리 추가
- 수동 백업 실행
- 사용 중 카테고리 삭제 방지

### 화면 9. 보조 설명

- 주 작업 파일 경로
- 결과 Markdown 경로
- 히스토리 파일 경로
- 생성 규칙 안내

## 4. 핵심 사용자 흐름

### 흐름 A. 준비 데이터 정리

1. 사람 관리에서 동행자 프로필을 등록한다
2. 차량 관리에서 자주 쓰는 차량 정보를 등록한다
3. 장비 관리와 외부 링크에서 반복 데이터를 정리한다

### 흐름 B. 새 계획 작성

1. 대시보드 또는 사이드 메뉴에서 `새 캠핑 계획` 을 누른다
2. 계획 폼에 날짜, 장소, 조건을 입력한다
3. 등록된 사람과 차량을 선택한다
4. 메모에는 사이트 특이사항, 출발 전 체크, 임시 메모를 적는다
5. 저장 후 분석을 실행한다

### 흐름 C. AI 보조 사용

1. 계획 페이지에서 현재 조건을 저장한다
2. AI 보조에 요청을 입력한다
3. 응답과 장비 액션 제안을 확인한다
4. 필요한 제안만 사용자가 반영한다

### 흐름 D. 장비 카테고리 관리

1. 카테고리 설정으로 이동한다
2. 반복 장비/소모품/출발 전 점검 중 섹션을 고른다
3. 카테고리 표시 이름을 수정하거나 새 카테고리를 추가한다
4. 필요하면 같은 화면에서 로컬 백업을 실행한다

### 흐름 E. 히스토리 아카이브

1. 캠핑이 끝난 계획에서 `히스토리로 이동` 을 실행한다
2. 계획 파일은 목록에서 사라진다
3. 히스토리에서 당시 동행자와 차량 스냅샷을 확인한다
4. 실제 메모를 보완한다

## 5. 상태 표시

- `idle`
- `loading`
- `success`
- `warning`
- `error`

분석 저장만 실패한 경우:

- Markdown 본문은 유지
- `warning` 상태로 저장 실패 안내

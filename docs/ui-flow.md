# UI 흐름

## 1. 문서 목적

이 문서는 현재 로컬 웹 UI의 주요 화면과 사용자 흐름을 정의한다.

## 2. 기본 원칙

- 첫 화면은 `대시보드`
- 메뉴는 장비, 계획, 히스토리, 외부 링크, 관리 설정을 분리한다
- 계획 페이지는 `폼 + AI 보조 + 분석 결과` 를 한 화면에서 다룬다
- AI 제안은 자동 반영하지 않는다

## 3. 주요 화면

### 화면 1. 대시보드

- 운영 현황 카드
- 예정 캠핑 목록
- 최근 히스토리 목록
- 빠른 이동 버튼

### 화면 2. 장비 관리

- 반복 장비 탭
- 소모품 탭
- 출발 전 점검 탭
- 목록 편집
- 항목 추가
- 카테고리 셀렉트 선택

### 화면 3. 캠핑 계획

- 계획 목록
- 계획 폼 편집
- 동행자 프로필 빠른 추가/수정
- AI 보조 대화
- 분석 실행
- 결과 Markdown
- 히스토리 아카이브

### 화면 4. 캠핑 히스토리

- 히스토리 목록
- 저장된 결과 Markdown 열기
- 실제 인원/메모 수정
- 삭제

### 화면 5. 외부 링크

- 카테고리별 링크 목록
- 카테고리 그룹 안에서 바로 수정
- 링크 추가/수정/삭제

### 화면 6. 관리 설정

- 장비 섹션별 카테고리 목록
- 카테고리 표시 이름 수정
- 새 카테고리 추가
- 사용 중 카테고리 삭제 방지

## 4. 핵심 사용자 흐름

### 흐름 A. 새 계획 작성

1. 대시보드 또는 사이드 메뉴에서 `새 캠핑 계획` 을 누른다
2. 계획 폼에 날짜, 장소, 조건과 동행자 ID를 입력한다
3. 없는 동행자 ID는 같은 화면에서 바로 등록하거나 저장 시 기본 프로필로 자동 추가한다
4. 저장 후 분석을 실행한다

### 흐름 B. AI 보조 사용

1. 계획 페이지에서 현재 조건을 저장한다
2. AI 보조에 요청을 입력한다
3. 응답과 장비 액션 제안을 확인한다
4. 필요한 제안만 사용자가 반영한다

### 흐름 C. 장비 보강 후 재분석

1. 장비 관리에서 부족한 수량이나 점검 상태를 수정한다
2. 계획 페이지로 돌아간다
3. 분석을 다시 실행한다

### 흐름 C-1. 장비 카테고리 관리

1. 관리 설정으로 이동한다
2. 반복 장비/소모품/출발 전 점검 중 섹션을 고른다
3. 카테고리 표시 이름을 수정하거나 새 카테고리를 추가한다
4. 영문 카테고리 코드를 직접 입력한다
5. 장비 관리 화면에서 새 카테고리를 선택한다

### 흐름 D. 히스토리 아카이브

1. 캠핑이 끝난 계획에서 `히스토리로 이동` 을 실행한다
2. 계획 파일은 목록에서 사라진다
3. 히스토리에서 실제 인원과 메모를 정리한다

## 5. 상태 표시

- `idle`
- `loading`
- `success`
- `warning`
- `error`

분석 저장만 실패한 경우:

- Markdown 본문은 유지
- `warning` 상태로 저장 실패 안내

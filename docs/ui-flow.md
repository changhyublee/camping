# UI 흐름

## 1. 문서 목적

이 문서는 현재 로컬 웹 UI의 주요 화면과 사용자 흐름을 정의한다.

## 2. 기본 원칙

- 첫 화면은 `대시보드`
- 메뉴는 `운영 허브 / 준비 데이터 / 관리 설정` 세 그룹으로 나눈다
- 모든 페이지는 `page intro + page-level tabs` 를 기본 구조로 삼고, 필요한 화면만 내부 상세 탭을 추가한다
- 계획 페이지는 페이지 탭 `계획 목록`, `원본 입력`, `AI·결과` 로 나눈다
- 사람과 차량은 계획에서 직접 입력하지 않고 준비 데이터 화면에서 관리한 뒤 선택한다
- AI 제안은 자동 반영하지 않는다
- 분석 실행은 백그라운드 작업으로 시작하고 pending 작업 동안만 SSE 실시간 이벤트로 화면을 갱신한다
- 반복 장비 메타데이터 수집도 백그라운드 작업으로 시작하고 pending 작업 동안만 SSE 실시간 이벤트로 아이템별 상태를 갱신한다
- 히스토리 후기/회고 저장도 백그라운드 개인화 학습으로 이어지고 pending 작업 동안만 SSE 실시간 이벤트로 상태를 갱신한다
- SSE 연결은 pending 분석, 메타데이터 수집, 개인화 학습 작업이 있을 때만 유지하고, 모두 idle이면 닫는다

## 3. 주요 화면

### 화면 1. 대시보드

- 페이지 탭 `운영 요약`, `빠른 실행`, `최근 기록`
- 운영 현황 카드
- 예정 캠핑 목록
- 최근 히스토리 목록
- 빠른 이동 버튼

### 화면 2. 사람 관리

- 페이지 탭 `사람 목록`, `프로필 편집`
- 사람 목록
- 이름, 연령대, 출생연도 입력
- 건강 특이사항, 필수 복용약, 민감도 수정

### 화면 3. 차량 관리

- 페이지 탭 `차량 목록`, `차량 편집`
- 차량 목록
- 표시 이름과 설명
- 탑승 인원, 적재량, 차량 메모 수정

### 화면 4. 장비 관리

- 페이지 탭 `장비 목록`, `상세 작업`
- 반복 장비 탭
- 소모품 탭
- 출발 전 점검 탭
- `상세 작업` 탭 상단 요약 스트립
- `상세 작업` 내부 탭 `작업 요약`, `항목 추가`
- 카테고리 그룹 기본 접힘
- 아이템 상세 기본 접힘
- 목록 편집
- 항목 추가
- 카테고리 셀렉트 선택
- 반복 장비 상세 패널에서 모델명, 구매 링크, AI 메타데이터, 재수집 버튼 표시
- 반복 장비는 아이템별 `메타데이터 수집 중...`, `메타 실패`, `메타 중단` 상태를 독립 표시
- 장비 페이지 재진입 후에도 현재 메타데이터 작업 상태를 다시 읽어 복원

### 화면 5. 캠핑 계획

- 페이지 탭 `계획 목록`, `원본 입력`, `AI·결과`
- 계획 목록
- 계획 폼 편집
- 제목 위 `날씨 자동 수집` 카드와 `수집` 버튼
- 등록된 동행자 다중 선택
- 등록된 차량 단일 선택
- 선택된 사람/차량 요약
- 선택된 동행자 summary 카드에서 메일 발송 대상 체크
- `AI·결과` 탭 상단 요약 카드
- `AI·결과` 내부 상세 탭 `분석 결과`, `AI 보조`, `누적 학습`
- `분석 결과` 상세 탭에서 현재 선택 수신자 요약과 수동 메일 발송 버튼 표시
- 누적 학습 요약 카드
- AI 보조 대화
- 분석 실행
- 분석 상태 배너와 `분석 중...` 버튼
- 전체 분석 완료 후 선택 동행자 메일 발송
- 결과 Markdown
- 히스토리 아카이브

### 화면 6. 캠핑 히스토리

- 페이지 탭 `히스토리 목록`, `상세 보기`
- 히스토리 목록
- `상세 보기` 상단 요약 카드
- `상세 보기` 내부 탭 `요약`, `기록/결과`, `후기 작성`, `학습`
- 저장된 결과 Markdown 열기
- 당시 동행자/차량 스냅샷 확인
- 메모 수정
- 후기/회고 입력
- 회고 엔트리 목록
- 이번 캠핑에서 AI가 배운 점 카드
- 전역 개인화 학습 요약 카드
- 삭제

### 화면 7. 외부 링크

- 페이지 탭 `링크 목록`, `새 링크`
- 카테고리별 링크 목록
- 카테고리 그룹 안에서 바로 수정
- 링크 추가/수정/삭제

### 화면 8. 카테고리 설정

- 페이지 탭 `카테고리 목록`, `보조 작업`
- 장비 섹션별 카테고리 목록
- `보조 작업` 탭 상단 요약 스트립
- `보조 작업` 내부 탭 `새 카테고리`, `관리 원칙`, `로컬 백업`
- 카테고리 편집 카드 기본 접힘
- 카테고리 표시 이름 수정
- 새 카테고리 추가
- 수동 백업 실행
- 사용 중 카테고리 삭제 방지

### 화면 9. 보조 설명

- 페이지 탭 `파일 안내`, `운영 메모`
- 주 작업 파일 경로
- 결과 Markdown 경로
- 히스토리 파일 경로
- 생성 규칙 안내

## 4. 핵심 사용자 흐름

### 흐름 A. 준비 데이터 정리

1. 사람 관리에서 동행자 프로필을 등록한다
2. 차량 관리에서 자주 쓰는 차량 정보를 등록한다
3. 장비 관리와 외부 링크에서 반복 데이터를 정리한다
4. 반복 장비를 저장하거나 재수집하면 메타데이터는 백그라운드에서 계속되고, 상세 카드와 목록 배지로 현재 상태를 확인한다

### 흐름 B. 새 계획 작성

1. 대시보드 또는 사이드 메뉴에서 `새 캠핑 계획` 을 누른다
2. `계획 목록` 탭에서 새 계획 작성으로 들어간 뒤 `원본 입력` 탭에서 날짜, 장소, 조건을 입력한다
3. 제목 위 `날씨 자동 수집` 카드의 `수집` 버튼으로 Google 검색 기반 날씨 요약을 먼저 채우거나, 그대로 직접 입력한다
4. 등록된 사람과 차량을 선택한다
5. 메모에는 사이트 특이사항, 출발 전 체크, 임시 메모를 적는다
6. 날씨를 비워 둔 채 저장하면 서버가 저장 직후 백그라운드에서 자동 수집을 시작하고 완료 시 계획 파일에 반영한다
7. 저장 후 필요하면 `AI·결과` 탭에서 누적 학습 요약을 확인한다
8. `원본 입력` 탭에서 분석을 실행한다
9. 버튼이 `분석 중...` 으로 바뀌면 완료까지 기다리고 `AI·결과` 탭에서 결과 Markdown 자동 갱신을 확인한다
10. `원본 입력` 탭의 동행자 summary 카드에서 발송 대상을 체크한다
11. 전체 분석이 끝나면 `원본 입력` 또는 `AI·결과 > 분석 결과` 탭의 `분석 결과 메일 발송` 버튼으로 필요한 사람에게만 보낸다

### 흐름 C. AI 보조 사용

1. 계획 페이지에서 현재 조건을 저장한다
2. AI 보조에 요청을 입력한다
3. 응답과 장비 액션 제안을 확인한다
4. 필요한 제안만 사용자가 반영한다

### 흐름 D. 장비 카테고리 관리

1. 카테고리 설정으로 이동한다
2. `카테고리 목록` 탭에서 반복 장비/소모품/출발 전 점검 중 섹션을 고른다
3. 카테고리 표시 이름을 수정하거나, `보조 작업` 탭에서 새 카테고리를 추가한다
4. 필요하면 같은 화면의 `보조 작업 > 로컬 백업` 에서 백업을 실행한다

### 흐름 E. 히스토리 아카이브

1. 캠핑이 끝난 계획에서 `히스토리로 이동` 을 실행한다
2. 계획 파일은 목록에서 사라진다
3. 히스토리 `상세 보기` 탭에서 당시 동행자와 차량 스냅샷을 확인한다
4. `후기 작성` 탭에서 실제 메모와 후기/회고를 보완한다
5. 개인화 학습 상태가 `학습 업데이트 중...` 으로 바뀌고 완료 후 학습 요약이 갱신되는지 확인한다

단, 분석이 `queued` 또는 `running` 상태이면 `히스토리로 이동` 과 `계획 삭제` 는 비활성화된다.

## 5. 상태 표시

운영 배너 상태:

- `idle`
- `loading`
- `success`
- `warning`
- `error`

계획 분석 상태:

- `idle`
- `queued`
- `running`
- `completed`
- `failed`
- `interrupted`

분석 상태 표현 원칙:

- `queued` 또는 `running` 이면 `분석 중...` 버튼과 정보 배너를 유지한다
- 섹션 하나라도 추가 완료되면 최신 Markdown을 자동으로 다시 불러온다
- `failed` 또는 `interrupted` 이면 기존 Markdown은 유지하고 상태 배너만 갱신한다
- pending 분석이 없으면 계획 화면은 실시간 스트림을 유지하지 않고, 다음 재진입이나 새 분석 시작 시 상태 조회 API로 다시 맞춘다
- 새로고침 후와 SSE 재연결 직후에는 상태 조회 API로 현재 분석 상태를 복원한다

반복 장비 메타데이터 상태:

- `idle`
- `queued`
- `running`
- `failed`
- `interrupted`

메타데이터 상태 표현 원칙:

- `queued` 또는 `running` 이면 해당 아이템 버튼만 `메타데이터 수집 중...` 으로 바꾸고 비활성화한다
- 여러 반복 장비가 동시에 수집 중이어도 각 아이템 상태를 독립적으로 유지한다
- 수집 성공 이벤트를 받으면 최신 장비 데이터를 다시 불러와 상세 카드와 배지를 즉시 갱신한다
- `failed` 또는 `interrupted` 면 기존 메타데이터는 유지하고 경고와 재시도 가능 상태를 함께 보여 준다
- pending 메타데이터 작업이 없으면 장비 화면은 실시간 스트림을 닫고, 다음 재진입이나 재수집 시작 시 상태 조회 API로 다시 맞춘다
- 페이지 재진입 후와 SSE 재연결 직후에는 상태 조회 API로 현재 메타데이터 작업 상태를 복원한다

개인화 학습 상태:

- `idle`
- `queued`
- `running`
- `completed`
- `failed`
- `interrupted`

개인화 학습 상태 표현 원칙:

- 회고 저장 직후에는 히스토리 상세와 계획 화면에서 `학습 업데이트 중...` 상태를 즉시 보여 준다
- `completed` 이면 이번 캠핑 학습 카드와 전역 개인화 학습 요약 카드를 다시 읽어 갱신한다
- `failed` 또는 `interrupted` 면 마지막 성공한 학습 요약은 유지하고 경고 배너만 갱신한다
- pending 개인화 학습이 없으면 실시간 스트림을 닫고, 다음 회고 저장이나 페이지 재진입 시 조회 API로 상태를 복원한다
- 페이지 재진입 후와 SSE 재연결 직후에는 `GET /api/user-learning` 과 현재 선택된 히스토리 학습 조회 API로 상태와 결과를 복원한다

# 메뉴 구조

## 1. 문서 목적

이 문서는 사용자가 반복적으로 쓰기 편한 `운영형 로컬 캠핑 관리자` 기준의 메뉴 구조를 정의한다.

목표:

- 계획 실행 화면과 기준 데이터 화면의 책임을 분리한다
- 사람, 차량, 장비, 링크처럼 반복 관리되는 데이터를 준비 데이터로 묶는다
- 설명성 정보는 메인 작업 화면이 아니라 별도 보조 설명 화면으로 분리한다

## 2. 최상위 메뉴 그룹

현재 화면은 9개 페이지를 아래 3개 그룹으로 묶는다.

### 2.1 운영 허브

포함 메뉴:

- 대시보드
- 캠핑 계획
- 캠핑 히스토리

구성 원칙:

- 현재 상태 확인과 실제 실행 흐름을 한 묶음으로 둔다
- 사용자가 가장 자주 오가는 화면을 가장 먼저 노출한다
- 각 메뉴에는 이름만 아니라 역할 설명과 상태 메타를 함께 표시한다

### 2.2 준비 데이터

포함 메뉴:

- 사람 관리
- 차량 관리
- 장비 관리
- 외부 링크

구성 원칙:

- 계획에서 반복 참조하는 기준 데이터를 한 묶음으로 분리한다
- 계획 화면에서는 선택과 확인에 집중하고, 상세 CRUD는 준비 데이터 화면에서 처리한다

### 2.3 관리 설정

포함 메뉴:

- 카테고리 설정
- 보조 설명

구성 원칙:

- 장비 카테고리와 백업 같은 운영 기준값은 별도 섹션으로 둔다
- 파일 경로, 생성 규칙 같은 설명성 정보는 보조 설명 화면으로 격리한다

## 3. 화면별 책임

### 3.1 대시보드

목적:

- 앱 첫 진입 허브
- 운영 현황을 빠르게 확인

핵심 요소:

- 예정된 캠핑 계획 수
- 최근 히스토리 수
- 점검 필요/재고 부족 항목 수
- 외부 링크 수
- 빠른 이동 버튼

### 3.2 캠핑 계획

목적:

- 이번 캠핑 일정 작성과 분석 실행

핵심 요소:

- 계획 목록
- 새 계획 작성
- 날짜/장소/조건 폼
- 등록된 사람 다중 선택
- 등록된 차량 단일 선택
- 선택된 사람/차량 요약
- AI 보조 대화
- 분석 실행
- 결과 자동 저장
- 히스토리 아카이브

원칙:

- 폼이 원본 입력의 기준이다
- 사람과 차량은 여기서 새로 만들지 않고 준비 데이터에서 관리한 뒤 선택한다
- AI는 제안을 생성하지만 자동 저장하지 않는다

### 3.3 캠핑 히스토리

목적:

- 완료된 계획의 결과를 기록으로 남기고 재활용

핵심 요소:

- 일정/지역/인원/차량 요약
- 아카이브 당시 동행자 스냅샷
- 아카이브 당시 차량 스냅샷
- 메모 수정
- 결과 문서 참조
- 삭제

원칙:

- 히스토리는 `계획 종료 후 아카이브` 로 생성한다
- 당시 사람/차량 정보는 스냅샷으로 고정해 이후 기준 데이터가 바뀌어도 기록이 유지되게 한다

### 3.4 사람 관리

목적:

- 캠핑 인원 프로필 원본 데이터 관리

핵심 요소:

- 사람 목록
- 이름, 연령대, 출생연도
- 건강 특이사항
- 필수 복용약
- 추위/더위/비 민감도

### 3.5 차량 관리

목적:

- 반복 사용하는 차량 기준 데이터 관리

핵심 요소:

- 차량 목록
- 표시 이름과 설명
- 탑승 인원
- 적재량
- 차량별 메모

### 3.6 장비 관리

목적:

- 보유 장비와 출발 전 점검 항목 관리

하위 섹션:

- 반복 장비
- 소모품
- 출발 전 점검

핵심 액션:

- 카테고리별 목록 보기
- 항목 추가
- 항목 수정
- 수량 변경
- 상태 변경
- 삭제
- 카테고리 셀렉트 사용

### 3.7 외부 링크

목적:

- 날씨, 장소, 맛집, 장보기 등 참고 링크를 직접 관리

핵심 요소:

- 카테고리별 링크 목록
- 링크 추가/수정/삭제
- 새 탭 열기

### 3.8 카테고리 설정

목적:

- 장비 섹션별 관리 기준값과 운영 데이터 백업을 관리

핵심 요소:

- 수동 백업 실행
- 반복 장비/소모품/출발 전 점검 카테고리 목록
- 카테고리 표시 이름 수정
- 새 카테고리 추가/삭제
- 카테고리 코드 확인

### 3.9 보조 설명

목적:

- 파일 경로와 생성 규칙 같은 설명성 정보를 한곳에 모아 제공

핵심 요소:

- 주 작업 파일 경로
- 결과 Markdown 경로
- 히스토리 파일 경로
- 계획 저장/분석 실행/히스토리 이동 시 생성 규칙

## 4. 기본 사용자 흐름

### 흐름 A. 기준 데이터 준비

1. 사람 관리에서 동행자 프로필을 등록한다
2. 차량 관리에서 자주 쓰는 차량 정보를 등록한다
3. 장비 관리와 외부 링크에서 반복 데이터를 정리한다

### 흐름 B. 계획 만들기

1. 대시보드 또는 캠핑 계획에서 새 계획을 만든다
2. 날짜, 장소, 조건을 폼에 입력한다
3. 등록된 사람과 차량을 선택한다
4. AI 보조 제안을 검토한 뒤 필요한 항목만 직접 반영한다
5. 분석을 실행해 결과를 확인한다

### 흐름 C. 캠핑 종료 후 정리

1. 계획 페이지에서 히스토리 아카이브를 실행한다
2. 히스토리에서 당시 메모를 정리한다
3. 다음 계획에 참고할 결과 문서를 다시 연다

## 5. 데이터 연결 기준

- `사람 관리` -> `companions.yaml`
- `차량 관리` -> `vehicles.yaml`
- `장비 관리` -> `equipment/*.yaml`
- `카테고리 설정` -> `equipment/categories.yaml`
- `캠핑 계획` -> `trips/*.yaml`
- `캠핑 히스토리` -> `history/*.yaml`
- `외부 링크` -> `links.yaml`
- `분석 결과` -> `outputs/*.md`

## 6. 구현 기준

- 첫 화면은 `대시보드`
- 네비게이션은 메뉴를 항상 드러내는 구조를 우선
- 메뉴는 `운영 허브`, `준비 데이터`, `관리 설정` 그룹으로 구분해 보여준다
- 계획 화면은 선택과 실행에 집중하고, 사람/차량 CRUD는 별도 화면으로 분리한다
- 보조 설명 화면에만 경로와 생성 규칙을 노출한다
- 브라우저는 OpenAI API를 직접 호출하지 않는다
